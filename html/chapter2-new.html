2.1 基于符号与逻辑的早期智能体

人工智能领域的早期探索，深受数理逻辑和计算机科学基本原理的影响。在那个时代，研究者们普遍持有一种信念：人类的智能，尤其是逻辑推理能力，可以被形式化的符号体系所捕捉和复现。这一核心思想催生了人工智能的第一个重要范式——符号主义（Symbolicism），也被称为"逻辑AI"或"传统AI"。

2.1.1 物理符号系统假说

图 2.2 物理符号系统的构成元素

充分性论断

任何一个物理符号系统，都具备产生通用智能行为的充分手段。

必要性论断

任何一个能够展现通用智能行为的系统，其本质必然是一个物理符号系统。

2.2 构建基于规则的聊天机器人

在探讨了符号主义的理论挑战后，本节我们将通过一个具体的编程实践，来直观地感受基于规则的系统是如何工作的。我们将尝试复现人工智能历史上一个极具影响力的早期聊天机器人——ELIZA。

2.2.1 ELIZA 的设计思想

基本信息

  • 开发者：约瑟夫·魏泽鲍姆（Joseph Weizenbaum）
  • 时间：1966年
  • 机构：麻省理工学院
  • 脚本：DOCTOR（罗杰斯学派心理治疗师）

工作原理

  • • 识别用户输入中的关键词
  • • 应用预设的转换规则
  • • 将陈述转化为开放式提问
  • • 营造"智能"和"共情"的假象

魏泽鲍姆的设计思想并非要创造一个真正能够"理解"人类情感的智能体，恰恰相反，他想证明的是，通过一些简单的句式转换技巧，机器可以在完全不理解对话内容的情况下，营造出一种"智能"和"共情"的假象。

2.2.3 核心逻辑的实现 

import re
import random

# 定义规则库：模式(正则表达式) -> 响应模板列表
rules = {
    r'I need (.*)': [
        "Why do you need {0}?",
        "Would it really help you to get {0}?",
        "Are you sure you need {0}?"
    ],
    r'Why don\'t you (.*)\?': [
        "Do you really think I don't {0}?",
        "Perhaps eventually I will {0}.",
        "Do you really want me to {0}?"
    ],
    r'Why can\'t I (.*)\?': [
        "Do you think you should be able to {0}?",
        "If you could {0}, what would you do?",
        "I don't know -- why can't you {0}?"
    ],
    r'I am (.*)': [
        "Did you come to me because you are {0}?",
        "How long have you been {0}?",
        "How do you feel about being {0}?"
    ],
    r'.* mother .*': [
        "Tell me more about your mother.",
        "What was your relationship with your mother like?",
        "How do you feel about your mother?"
    ],
    r'.* father .*': [
        "Tell me more about your father.",
        "How did your father make you feel?",
        "What has your father taught you?"
    ],
    r'.*': [
        "Please tell me more.",
        "Let's change focus a bit... Tell me about your family.",
        "Can you elaborate on that?"
    ]
}

# 定义代词转换规则
pronoun_swap = {
    "i": "you", "you": "i", "me": "you", "my": "your",
    "am": "are", "are": "am", "was": "were", "i'd": "you would",
    "i've": "you have", "i'll": "you will", "yours": "mine",
    "mine": "yours"
}

def swap_pronouns(phrase):
    """
    对输入短语中的代词进行第一/第二人称转换
    """
    words = phrase.lower().split()
    swapped_words = [pronoun_swap.get(word, word) for word in words]
    return " ".join(swapped_words)

def respond(user_input):
    """
    根据规则库生成响应
    """
    for pattern, responses in rules.items():
        match = re.search(pattern, user_input, re.IGNORECASE)
        if match:
            # 捕获匹配到的部分
            captured_group = match.group(1) if match.groups() else ''
            # 进行代词转换
            swapped_group = swap_pronouns(captured_group)
            # 从模板中随机选择一个并格式化
            response = random.choice(responses).format(swapped_group)
            return response
    # 如果没有匹配任何特定规则，使用最后的通配符规则
    return random.choice(rules[r'.*'])

# 主聊天循环
if __name__ == '__main__':
    print("Therapist: Hello! How can I help you today?")
    while True:
        user_input = input("You: ")
        if user_input.lower() in ["quit", "exit", "bye"]:
            print("Therapist: Goodbye. It was nice talking to you.")
            break
        response = respond(user_input)
        print(f"Therapist: {response}")

对话示例

You:

Therapist:

You:

Therapist:

根本局限性分析

缺乏语义理解

系统不理解词义，无法处理否定词等语义信息，容易产生语义不通的回应

无上下文记忆

系统是无状态的，每次回应仅基于当前单句输入，无法进行连贯的多轮对话

扩展性问题

增加更多规则导致规则库规模爆炸，规则间冲突管理变得极其复杂

ELIZA效应：尽管存在这些局限性，许多用户仍相信ELIZA能理解自己，这种智能的幻觉主要源于其巧妙的对话策略以及人类天生的情感投射心理。

2.3 马文·明斯基的心智社会

符号主义的探索和ELIZA的实践，共同指向了一个问题：通过预设规则构建的、单一的、集中的推理引擎，似乎难以通向真正的智能。马文·明斯基（Marvin Minsky）在他的《心智社会》（The Society of Mind）一书中提出了一个革命性的观点。

What magical trick makes us intelligent? The trick is that there is no trick. The power of intelligence stems from our vast diversity, not from any single, perfect principle.

— Marvin Minsky

2.3.1 对单一整体智能模型的反思

核心问题

20世纪70至80年代，符号主义的局限性日益明显。专家系统缺乏儿童般的常识；SHRDLU无法理解积木世界之外的任何事情；ELIZA对对话内容本身一无所知。

"理解"是什么？

是单一能力还是多种心智过程的协同工作？

"常识"是什么？

是逻辑规则库还是分布式经验网络？

如何构建智能体？

统一逻辑系统还是多样化的组件协作？

2.3.2 作为协作体的智能

图 2.6 "心智社会"中搭建积木塔行为的涌现机制示意图

简单智能体

极其简单的、专门化的心智过程，自身是"无心"的：

  • • LINE-FINDER：识别线条的智能体
  • • GRASP：负责抓握的智能体
  • • SEE：视觉处理的智能体
  • • PUT：放置操作的智能体

机构（Agency）

一组协同工作的智能体，形成更复杂的系统：

  • • BUILD-TOWER：搭建塔的高层机构
  • • BUILDER：循环执行的机构
  • • ADD-BLOCK：添加积木的机构
  • • FIND-BLOCK：寻找积木的机构

涌现（Emergence）

复杂的、有目的性的智能行为，并非由某个高级智能体预先规划，而是从大量简单的底层智能体之间的局部交互中自发产生的。没有一个智能体拥有全局规划，但整体行为却表现出高度智能。

2.3.3 对多智能体系统的理论启发

心智社会理论为分布式人工智能（DAI）和多智能体系统（MAS）提供了重要的概念基础。如果一个心智内部的智能可以通过大量简单智能体的协作而涌现，那么在多个独立的计算实体之间，是否也能通过协作涌现出更强大的"群体智能"？

去中心化控制

不存在中央控制器，设计无中心节点的协调机制和任务分配策略

涌现式计算

复杂问题解决方案从简单的局部交互规则中自发产生

智能体的社会性

研究智能体间的通信语言、交互协议、协商策略等

2.4 学习范式的演进与现代智能体

前文探讨的"心智社会"理论，在哲学层面为群体智能和去中心化协作指明了方向，但实现路径尚不明确。与此同时，符号主义在应对真实世界复杂性时暴露的根本性挑战也表明仅靠预先编码的规则无法构建真正鲁棒的智能。这一设问开启了人工智能的"学习"时代。

2.4.2 基于强化学习的智能体

图 2.8 强化学习的核心交互循环

AlphaGo案例

AlphaGo通过自我对弈学习围棋，观察棋盘状态（环境状态），决定落子位置（行动），根据胜负结果获得奖励信号，通过数百万次自我对弈不断调整内部策略。

核心要素

  • 智能体（Agent）

    学习者和决策者

  • 环境（Environment）

    智能体外部的一切

  • 状态（State, S）

    环境的特定描述

  • 行动（Action, A）

    智能体能采取的操作

  • 奖励（Reward, R）

    评价行动好坏的信号

工作模式

  1. 1

    智能体观察到环境当前状态 $S_t$

  2. 2

    基于策略 $\pi$ 选择行动 $A_t$ 并执行

  3. 3

    环境转移到新状态 $S_{t+1}$

  4. 4

    环境反馈即时奖励 $R_{t+1}$

  5. 5

    智能体利用反馈优化策略
